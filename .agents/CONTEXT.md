# TaBortMig — Domain Context

TaBortMig is a small web app that helps users keep track of when they have opted
out from Swedish people-search sites and sends reminder emails before those
opt-outs may need to be renewed.

MVP v1 focuses on:
- Showing a curated list of supported sites with links to their opt-out pages.
- Letting users register with an email address.
- For each supported site, letting users mark that they have opted out and
  enter the date when they did so.
- Storing a per-site default duration and sending a reminder email one week
  before that duration expires.

## Domain language

- `Site`
  - A Swedish people-search or data-aggregator site supported by the app
    (e.g., Ratsit, MrKoll, Hitta, Merinfo, Eniro).
  - Has a human-readable name, a URL to the relevant opt-out/help page, and a
    default opt-out duration in months.

- `User`
  - A person using TaBortMig, identified primarily by an email address.
  - May have zero or more opt-out entries, one per supported site.

- `OptOutEntry`
  - A record that a given `User` has opted out from a given `Site`.
  - Includes at least:
    - `siteId` (reference to `Site`)
    - `optOutDate` (the date the user says they opted out)
    - derived `expiryDate` (optOutDate + site default duration)

- `Reminder`
  - A conceptual event that occurs when an `OptOutEntry` is approaching its
    `expiryDate`.
  - In implementation, reminders are generated by a scheduled backend job that
    sends an email to the `User` one week before `expiryDate`.

## Views (high-level)

- `HomePage`
  - Explains what TaBortMig does: helps track opt-outs and sends reminders.
  - Provides a simple call-to-action to start or sign in.

- `SitesPage`
  - Shows the list of supported `Site` entries with:
    - A link to the site's opt-out/help page.
    - A checkbox or toggle to indicate that the user has opted out.
    - A date input where the user can enter the opt-out date.

- `AccountPage` (or similar)
  - Shows the current user's email and their saved opt-out entries.
  - Allows updating opt-out dates if needed.

Details of the final route structure and additional views will be refined during
the MVP planning, but this captures the core domain concepts and flows.
